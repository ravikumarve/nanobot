# Contributing to nanobot

🎉 **Thank you for your interest in contributing to nanobot!** We welcome all contributions from the community.

## ⭐ First Things First

If you find nanobot useful, please consider giving it a star on GitHub! Stars help us:
- 📈 Increase visibility in the open-source community
- 🚀 Attract more contributors and users
- 💪 Motivate continued development

## 🤝 Ways to Contribute

### 1. 🐛 Report Bugs
- Use the [bug report template](.github/ISSUE_TEMPLATE/bug_report.md)
- Provide detailed steps to reproduce
- Include error logs and environment details

### 2. 🚀 Suggest Features
- Use the [feature request template](.github/ISSUE_TEMPLATE/feature_request.md)
- Explain the problem and your proposed solution
- Consider if it aligns with project goals

### 3. 💻 Code Contributions
- Pick an issue from the roadmap or open issues
- Follow our coding standards
- Add tests for new functionality
- Update documentation as needed

### 4. 📚 Improve Documentation
- Fix typos and clarify explanations
- Add examples and usage guides
- Translate documentation to other languages

### 5. 🔍 Review Code
- Help review pull requests
- Provide constructive feedback
- Ensure code quality and standards

## 🛠️ Development Setup

### Prerequisites
- Python 3.11+
- Git
- Basic understanding of async programming

### Installation
```bash
# Fork and clone the repository
git clone https://github.com/your-username/nanobot.git
cd nanobot

# Install in development mode
pip install -e .[dev]

# Run tests to ensure everything works
pytest
```

## 📝 Pull Request Process

1. **Fork** the repository
2. **Create a branch** for your feature (`git checkout -b feature/amazing-feature`)
3. **Commit your changes** (`git commit -m 'Add amazing feature'`)
4. **Push to the branch** (`git push origin feature/amazing-feature`)
5. **Open a Pull Request** with a clear description

### PR Requirements
- ✅ Tests pass (`pytest`)
- ✅ Code follows existing style (`ruff check .`)
- ✅ Documentation updated if needed
- ✅ Description explains the changes and motivation

## 🎯 Code Standards

### Python Style
- Follow [PEP 8](https://pep8.org/) guidelines
- Use type hints for all function parameters and return values
- Maximum line length: 100 characters
- Use f-strings for string formatting

### Import Organization
1. Standard library imports
2. Third-party imports
3. Local application imports

### Documentation
- Use Google-style docstrings
- Document all public functions and classes
- Include examples for complex functionality

## 🐛 Bug Reports

When reporting bugs, please include:
- **Steps to reproduce** the issue
- **Expected behavior** and **actual behavior**
- **Environment details** (OS, Python version, etc.)
- **Error messages** and **logs**
- **Screenshots** if applicable

## 💡 Feature Requests

For feature requests, please:
- Explain the **problem** you're trying to solve
- Describe your **proposed solution**
- Provide **use cases** and examples
- Consider **alternative approaches**

## 🏆 Recognition

All contributors will be:
- Listed in the project's README (via GitHub contributors)
- Acknowledged in release notes
- Given credit for their contributions
